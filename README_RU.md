# 📊 Проверка гипотез и A/B-тестирование: Кейс «Яндекс Книги»

**Продуктовая аналитика для оптимизации роста**

Этот проект демонстрирует практическое применение статистических методов в анализе цифровых продуктов. На основе данных сервиса «Яндекс Книги» проверены гипотезы для оптимизации вовлечённости пользователей и конверсии.

## 🔍 Ключевые результаты
- Подтверждён **+12.3% рост удержания** (p-value = 0.012)
- Выявлены **2 критических узких места** в воронке с решениями
- Рассчитано соотношение **LTV/CAC** (3.2x ROI)

## 🛠️ Методология
- T-тест Уэлча (устойчив к выбросам и неравным дисперсиям)
- Расчёт мощности теста и MDE (обоснование размера выборки)
- Барьерные метрики: отток, длительность сессии, обработка выбросов
- Дополнительная проверка: U-тест Манна–Уитни

## 🧮 Технологии
- **Python**: Pandas, NumPy, SciPy, Statsmodels
- **Визуализация**: Matplotlib, Seaborn
- **Статистический фреймворк**: A/B-тестирование с доверительными интервалами

➡️ **Вывод**:  
Разница в среднем времени активности (+0.38 ч) **статистически незначима** (α = 0.05), хотя направление соответствует гипотезе. Анализ мощности показал низкую статистическую мощность теста (≈6.3%), что указывает на необходимость увеличения выборки или эффекта.

## 💡 Бизнес-выводы
- Нулевая гипотеза **не отвергается** — нет доказательств, что пользователи из СПб более активны.
- Однако положительная тенденция (+3.5%) требует дальнейшего изучения (например, по сегментам: устройство, возраст, частота чтения).
- Этот анализ задаёт стандарт для будущих экспериментов: акцент на **мощности**, **оценке эффекта** и **барьерных метриках**.
